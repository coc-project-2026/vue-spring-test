01

要件定義

요건 정의
工程 1 / PHASE 1

고객의 보험 가입부터 보험금 지급까지의 업무를 디지털화하는 보험계약관리시스템(保険契約管理システム)을 구축한다. 본 장에서는 시스템이 만족해야 할 기능요건(機能要件)과 비기능요건(非機能要件)을 정의한다.

1.1 システム概要 시스템 개요

目的 (목적)保険契約のライフサイクル(見積→申込→引受→契約→請求→支払)を一元管理し、業務工数を削減する。
対象業務 (대상 업무)顧客管理 / 商品管理 / 契約管理 / 保険料計算 / 保険金請求・支払 / 帳票出力
利用者 (이용자)営業担当・引受担当(アンダーライター)・査定担当・顧客(マイページ)・管理者
稼働形態Web (社内業務システム + 顧客向けポータル) / 24時間365日 (오전 2~4시 보수 윈도우)

3.3 主要処理ロジック 주요 처리 로직

保険料計算 (自動車)

public Money calc(QuoteContext ctx) {
  RateTable rate = rateRepo.findByProduct(ctx.productId());
  Money base = rate.baseRate(ctx.coverageCode());
  // 年齢・等級・地域による係数を順次適用
  BigDecimal factor = ageFactor(ctx.age())
      .multiply(gradeFactor(ctx.noClaimGrade()))
      .multiply(areaFactor(ctx.area()));
  return base.multiply(factor).roundDown();  // 円未満切捨
}
